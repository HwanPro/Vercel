1. DESCRIPCIÓN DE LA ORGANIZACIÓN Y DEL SOFTWARE
================================================================================

PERFIL DE LA ORGANIZACIÓN:
-------------------------
Wolf Gym es un gimnasio local ubicado en Ica, Perú, que opera en el sector de 
fitness y bienestar. Se trata de una organización de tamaño mediano que atiende 
a una comunidad local de usuarios interesados en entrenamiento físico, 
musculación y acondicionamiento físico.

PROCESOS CLAVE:
- Gestión de membresías y suscripciones
- Control de acceso mediante sistema biométrico
- Venta de productos complementarios (agua, proteínas, suplementos)
- Seguimiento de rutinas de entrenamiento personalizadas
- Administración de pagos y deudas
- Comunicación con clientes vía WhatsApp

NECESIDAD O PROBLEMA A RESOLVER:
--------------------------------
El gimnasio necesitaba modernizar sus procesos operativos para:
- Automatizar el control de acceso mediante huellas dactilares, eliminando la 
  dependencia de tarjetas físicas
- Digitalizar la gestión de clientes y membresías
- Optimizar el seguimiento de asistencia y estadísticas de uso
- Centralizar la administración de productos, pagos y deudas
- Mejorar la experiencia del usuario con un sistema de rutinas personalizadas
- Facilitar la comunicación con los clientes

DESCRIPCIÓN FUNCIONAL DEL SISTEMA:
----------------------------------

MÓDULOS PRINCIPALES:

1. Sistema de Autenticación y Usuarios
   - Registro e inicio de sesión con roles (admin/cliente)
   - Gestión de perfiles de usuario
   - Autenticación de dos factores (2FA)

2. Sistema Biométrico
   - Captura y registro de huellas dactilares (hasta 10 dedos por usuario)
   - Verificación 1:1 y identificación 1:N
   - Integración con lector ZKTeco ZK9500

3. Control de Acceso y Asistencia
   - Check-in/check-out automático
   - Registro de asistencia en tiempo real
   - Dashboard de actividad en vivo
   - Historial de asistencia

4. Gestión de Membresías
   - Planes de suscripción (Mensual, Trimestral, Anual)
   - Control de vencimientos
   - Renovaciones automáticas
   - Seguimiento de días restantes

5. Sistema de Rutinas y Entrenamientos
   - Biblioteca de ejercicios con media (videos/imágenes)
   - Creación de rutinas personalizadas
   - Seguimiento de progreso con RPE (Rate of Perceived Exertion)
   - Sugerencias de progresión automática
   - Cálculo de 1RM (One Rep Max)

6. Gestión de Productos y Ventas
   - Catálogo unificado de productos del gimnasio
   - Control de inventario y stock
   - Sistema de deudas diarias
   - Integración con pasarela de pagos Culqi

7. Sistema de Pagos
   - Procesamiento de pagos con Culqi
   - Gestión de deudas y cobranza
   - Historial de transacciones
   - Notificaciones de vencimiento

8. Comunicación y Notificaciones
   - Integración con WhatsApp Business API (Twilio)
   - Notificaciones por email
   - Recordatorios de vencimiento

ACTORES/USUARIOS INVOLUCRADOS:
- Administradores: Gestión completa del sistema, clientes, productos y reportes
- Clientes: Acceso al gimnasio, consulta de rutinas, seguimiento de progreso
- Sistema Biométrico: Servicio independiente para procesamiento de huellas

FLUJO GENERAL DE USO:
1. Registro de Cliente: El administrador registra nuevos clientes con datos 
   personales y plan de membresía
2. Enrollment Biométrico: Se capturan las huellas dactilares del cliente
3. Acceso Diario: El cliente accede mediante huella dactilar para 
   check-in/check-out
4. Entrenamiento: El cliente consulta sus rutinas asignadas y registra sus 
   entrenamientos
5. Compra de Productos: Los clientes pueden adquirir productos del gimnasio
6. Gestión de Pagos: El sistema gestiona automáticamente vencimientos y 
   notificaciones

ARQUITECTURA Y TECNOLOGÍAS UTILIZADAS:
--------------------------------------

FRONTEND:
- Next.js 15.2.2 con TypeScript
- React 18.2.0 con hooks y context
- Tailwind CSS para estilos
- Radix UI para componentes accesibles
- React Hook Form para formularios
- SWR para gestión de estado del servidor

BACKEND:
- Next.js API Routes (serverless functions)
- NextAuth.js para autenticación
- Prisma ORM para base de datos
- PostgreSQL como base de datos principal

SERVICIOS EXTERNOS:
- Culqi para procesamiento de pagos
- Twilio WhatsApp API para notificaciones
- AWS S3 para almacenamiento de archivos
- Nodemailer para emails

SISTEMA BIOMÉTRICO:
- C# .NET 8 como servicio independiente
- ZKFinger SDK para integración con hardware
- Windows Service para ejecución en background

MODELO DE DESPLIEGUE:
- Híbrido: 
  * Frontend/Backend: Cloud (Vercel o similar)
  * Servicio Biométrico: On-premise (Windows Server en el gimnasio)
  * Base de datos: Cloud (PostgreSQL)

RESTRICCIONES IDENTIFICADAS:
----------------------------

DEPENDENCIAS TÉCNICAS:
- Hardware específico: Requiere lector ZKTeco ZK9500
- Sistema operativo: Windows para el servicio biométrico
- Conectividad: Internet estable para sincronización
- Navegador: Chrome/Edge recomendados para mejor compatibilidad

LIMITACIONES:
- Escalabilidad: El servicio biométrico está limitado a un dispositivo por 
  ubicación
- Conectividad: Dependencia de internet para operaciones críticas
- Hardware: Requiere mantenimiento del lector biométrico
- Personalización: Limitado a configuraciones predefinidas del SDK

RESUMEN EJECUTIVO
================================================================================

Wolf Gym es un sistema completo de gestión de gimnasio que automatiza los 
procesos operativos mediante:

- Sistema biométrico avanzado con lector ZKTeco ZK9500
- Gestión integral de clientes y membresías
- Sistema de rutinas personalizadas tipo Strong
- Control de acceso automatizado con check-in/check-out
- Gestión de productos y pagos integrada
- Comunicación automatizada vía WhatsApp

PUNTOS DESTACADOS:
1. Arquitectura Híbrida: Combina cloud (Next.js/PostgreSQL) con on-premise 
   (servicio biométrico C#)
2. Tecnologías Modernas: Next.js 15, TypeScript, Prisma, Tailwind CSS
3. Integraciones Robustas: Culqi (pagos), Twilio (WhatsApp), AWS S3
4. Sistema Biométrico Avanzado: Soporte para 10 dedos por usuario con 
   algoritmo Hamming distance
5. Plan SQA Completo: 8 semanas de testing con métricas específicas y 
   herramientas profesionales
